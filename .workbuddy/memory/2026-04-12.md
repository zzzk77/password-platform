- 需确保Docker后端服务停止后再启动本地后端

### 项目结构
- attack/: 攻击端前端 (Vue3 + Vite)
- defense/: 防御端前端 (Vue3 + Vite)
- shared/: 共享组件
- docker/: 各种靶场服务 (Flask)
- nginx/: Nginx网关配置
- 后端: NestJS 服务 (padp-nest-server，端口4001)

### 部署配置完成
- docker-compose.yml: 一键部署配置（nginx网关 + 3个古典密码靶场）
- Dockerfile.nginx: Nginx网关镜像（含前端构建）
- nginx/nginx.conf: 路由配置
- deploy.sh: 一键部署脚本
- 靶场requirements.txt: 升级Flask==3.0.0修复兼容性问题

### Nginx路由配置
- / → 攻击端前端
- /defense/ → 防御端前端
- /api/ → NestJS后端 (host.docker.internal:4001)
- /ctf/ → CTF入门靶场
- /comm/ → 古典通信靶场
- /crack/ → 古典破解靶场

### 新增靶场：DES弱密钥靶场 (对称加密)
- 路径: docker/range-des-weak/
- 端口: 8084
- 特点: 逼真的旧银行磁条卡系统界面
- 功能:
  - 模拟银行信用卡展示
  - 交易加密/解密工具
  - 密钥安全分析
  - 终端命令行界面
  - Flag提交功能
- API端点:
  - /des/encrypt - DES加密
  - /des/decrypt - DES解密
  - /des/double-encrypt - 二次加密测试
  - /des/check-weak-key - 弱密钥检测
  - /des/keyspace - 密钥空间信息
  - /des/ecb-analyze - ECB模式分析
- 挑战题目:
  - DES弱密钥识别: flag{BANKSAFE}
  - DES-ECB模式攻击: flag{TRANSFER}
  - DES密钥空间缩减: flag{48}
  - DES半弱密钥对: flag{FE01FE01}
